# OpenMeow.Lab 評価器v3・軌道研究

実施日: 2026-07-23  
対象: `default_mew`  
並列研究エージェント: 3

## 到達点

初回パイロットで見つかった「弱い接触でも高得点」「高速な短ストロークが有利」
「離脱後の揺れを測っていない」を評価器v2で修正しました。v2の並列研究でさらに、
押し込み速度を撫で速度へ混ぜていたことと、`durationSeconds` が軌道時間ではなく
目標を瞬間変更した後の待ち時間になっていたことを発見しました。

評価器v3と最終プロトコルでは次を実装しています。

- 目標部位の希望力から外れる接触を最終点へ乗算ゲートする。
- 明示した離脱だけを0.9秒測り、途中の接触切れを静止評価にしない。
- 速度を研究項目の `strokeAxis` 方向だけで測る。
- 未接触試行へ速度、反転、静止の加点を与えない。
- `durationSeconds` 全体で手の目標をsmoothstep補間し、長い動作を本当に遅くする。
- 標準ストローク時間を移動距離÷希望速度で決める。

この変更で組み込み自動探索の代表スコアは `39.07 → 54.09 → 66.19` へ改善しました。
その後、別部位への接触で連続性を稼げない目標固有判定へ厳格化した最終値は `64.02` です。
最終64候補探索では頭 `74.333`、頬 `83.556`、腕支持 `78.732`、手つなぎ
`72.947` となりました。頭と頬では速度自然度もそれぞれ `1.0`、`0.90447` まで
識別できています。

## 本体Naturalへの反映

4項目へ同じ設定を持ち込むmaximin比較では、次の研究候補が平均 `58.603`、
最低 `51.046` で最も崩れにくい結果でした。

| パラメータ | 値 |
|---|---:|
| ばね | 6.0 Hz |
| 減衰 | 1.10 |
| 速度上限 | 1.5 m/s |
| 加速度上限 | 14 m/s² |
| 先読み | 0.020 s |

これをOpenMeow本体の `Natural` プリセットへ採用しました。Labの接触
コンプライアンス `0.78` はDriverに対応物がないため書き込みません。
研究中は `/driver-profile/preview` だけを使い、ユーザーの実設定へは未適用です。

## データと限界

- `head-cheek-v3.*` — 軸速度、弱接触、高速動作の再検証
- `hand-limp-v3.*` — 支持、把持、未接触静止の再検証
- `generalization-v3.*` — 5候補×4項目の横断比較
- `trajectory-protocol.json` — smoothstep軌道版の最終256候補探索と固定候補比較

数値は決定論的なハーネス内の相対評価です。SteamVRでの人体IK、ゲームごとの差、
人が感じる快適さは未検証なので、次は実VRで `Legacy` と `Natural` のAB確認が必要です。
手つなぎの速度自然度はまだ0で、把持タスクをストローク型とは別の「接近後静止」
モデルで評価する余地があります。
